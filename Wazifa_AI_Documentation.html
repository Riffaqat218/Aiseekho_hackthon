Wazifa AI

The Intelligent, Multi-Agent Scholarship & Credential Ingestion Assistant

1. Executive Summary

Applying for competitive international and national academic scholarships (such as Fulbright, DAAD, Chevening, or HEC Pakistan) represents a severe logistical burden for students. Applicants must manually scour complex eligibility requirements, identify gaps in their credentials, draft highly specific recommendation request emails, generate calendars, and fill repetitive application portals.

Wazifa AI solves this end-to-end. By leveraging a high-performance multi-agent cognitive backend integrated with on-device OCR document parsing, Wazifa AI analyzes student metrics (CGPA, degree level, major) in real-time, builds a customized "Gap Radar" timeline, and autonomously executes necessary application tasks. It represents a state-of-the-art implementation of utility-based agent design, fully optimized for scale, speed, and clean cross-platform client accessibility.

Key Innovation: The Antigravity Action Engine

Instead of just recommending scholarships, Wazifa AI utilizes serverless AI actors to complete application tasks in parallel: auto-filling portals, drafting custom professor emails, plotting calendar sequence roadmaps, and composing Statement of Purpose (SOP) entries.

2. System Architecture

Wazifa AI's architecture is divided into three key layers: the client-side Flutter application, the NestJS middleware execution microservice, and the Supabase cloud datastore. Secret API keys are kept 100% backend-side to secure the runtime environment.

Plug-and-Play Physical Device Testing

To facilitate seamless local and production testing, we integrated a **Dynamic Backend Server Endpoint setting** directly inside the client application. Located at the bottom of the **Student Profile Section**, the user can instantly configure the server's endpoint:

• Local Wi-Fi Testing: Simply type your computer's local IP address (e.g. http://192.168.0.100:3000) to connect the physical Android phone directly to your local development server.
• Production Deployment Testing: Paste the secure GCP Cloud Run URL (https://wazifa-backend-514493666042.us-central1.run.app) to connect instantly to the cloud-hosted NestJS multi-agent system.

3. Multi-Agent Workflows (The Action Engine)

The cognitive processing inside Wazifa AI is handled by a sequence of highly specialized agents. When a student chooses a scholarship, the orchestrator executes a multi-stage workflow, streaming the intermediate agent "traces" to the UI to maintain transparency and auditability.

■ Ingestor & Matcher Agents

The ingestor parses documents using OCR and standardizes metrics, while the Matcher filters out candidates based on CGPA constraints (ensuring strict filtering boundaries) and local/foreign exclusion flags.

■ The Cognitive Action Core

Orchestrates tasks in parallel: building mock schemas, drafting custom emails referencing user CGPA/Major, setting milestone calendars with acquisition timelines, and compiling personalized SOP paragraphs.

5. Verification & Deployment Pipeline

To ensure maximum code quality and prevent runtime compile failures on the physical device, a fully automated **GitHub Actions CI/CD Pipeline** was established:

Automated Release Delivery Checklist:
  • Code Push Action: Every push to the main branch triggers the flutter-build-apk.yml workflow.
  • Flutter SDK Resolving: Fetches the latest stable Flutter SDK dynamically, preventing Dart package dependency conflicts.
  • Native Compilation: Automatically builds a production-ready, release-signed app-release.apk in under 9 minutes.
  • Artifact Distribution: Automatically uploads the compiled wazifa-release-apk release bundle artifact to GitHub for immediate testing and download.
GCP Cloud Run Hosting Setup

The NestJS backend was compiled into a lightweight production Docker container and deployed directly onto Google Cloud Run. Using serverless HTTPS endpoints, the backend handles real-time REST actions securely with 99.9% uptime.
